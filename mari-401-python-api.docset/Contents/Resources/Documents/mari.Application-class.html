autosaveTriggered ( self , ScheduledTime )

source code  

This is emitted when the application autosave is triggered.

We call the autosave manager several times at various places throughout the application. The autosave manager is responsible to maintain the current time and when the autosave should next happen and update the timer accordingly. This means that we could call the autosave manager a hundred times and only one call will get through at the scheduled time. This makes the autosaving efficient. To that end, if the autosave call is not completed (because it's not yet time), we return the scheduled time as a string. Otherwise, if the call does go through and an autosave is performed, we return an empty string here.

Parameters:
  • ScheduledTime (str) - This is the time when the autosave is scheduled next.
Returns: None

backgroundJobsRunning ( self , AreJobsRunning )

source code  

This is emitted when Mari begins and end background processing of jobs.

Mari can run multiple jobs in the background while the user is working. These can be tasks like exporting or importing images or performing long running processing. This signal informs the user when Mari starts or end processing background tasks. This is only emitted as True when Mari goes from a state where no background jobs are running to running background jobs. If Mari is currently running background jobs and new jobs are added this signal is not emitted. #Print out a message when background jobs start or stop

print 'Background Job Running ', a

mari.utils.connect(mari.app.backgroundJobsRunning, printRunning)

Parameters:
  • AreJobsRunning (bool) - True if background jobs are now scheduled, or False if there are no more background jobs
Returns: None

startProcessing ( self , Description , NumSteps = 100 , CanCancel = False )

source code  

Registers the start of a long processing operation.

Mari gives the user feedback on the status of long running processes. This method indicates the start of one of these processes. It is a good idea to always start this when processing data. Mari will display a busy icon, and calls to setProgress() will update a status bar in the bottom right corner of the application GUI.

Calls to startProcessing() must always be followed by a stopProcessing() call. It may also be useful to call processEvents() during your processing loop to give the application time to update the GUI.

This function will automatically call processEvents() once to ensure that the new status bar is displayed to the user.

Parameters:
  • Description (str) - A user-readable description of the process - e.g. "Processing Data". This will be displayed in the status bar.
  • NumSteps (int) - The number of steps to be performed in the processing. Calls to setProgress should indicate the number of steps completed.
  • CanCancel (bool) - Set this to True if the action being executed can be safely canceled
Returns: None

Attention: If run in the script editor, this function will cause Mari to become non-interactive until stopProcessing() is called, so make sure you call it as part of your script or you will not be able to enter any further commands.

See Also: suspendProcessing() , canceledProcessing() , wasProcessingCanceled() , setProgress() , setProgressDescription() , stopProcessing() , processEvents() , stepProgress() , numProcessingSteps()

suspendProcessing ( self )

source code  

Temporarily suspends a long processing operation.

Normally, long-running processing operations ignore mouse and keyboard input from the user while running. Use this function to suspend the processing temporarily, and allow mouse and keyboard input again, if input is required from the user - such as in the case of a message box querying the user for action.

Make sure you call resumeProcessing() again as soon as possible, or the status bar will not be updated, and users may be able to interfere with processing operations by making changes through the GUI at the same time.

Returns: None

See Also: resumeProcessing()

Checks to see if the user has chosen to cancel a long-running process.

Users can cancel many long-running processes by clicking the 'X' button on the progress bar when one is present, or by pressing the "Escape" key.
