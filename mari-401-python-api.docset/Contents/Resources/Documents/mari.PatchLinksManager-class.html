Class PatchLinksManager

source code 

object --+
         |
        PatchLinksManager

Handles patch image linking.

Using this manager, you can create and managed named groups of linked images. Linked patches share their underlying image contents, so if you paint on one linked patch, all other linked patches are updated.

Currently, the API restricts linkage to Image objects on the same ImageSet . At some point in the future, which restriction will be removed. Some methods require an seemingly redundant ImageSet parameter to make provision for such a change.

Example Code 

>>> # This example creates a patch link between 1001 and 1011
>>> import mari
>>> geo = mari.geo.current()
>>> patch_1001 = geo.patch(0)
>>> patch_1011 = geo.patch(10)
>>> layer = geo.currentChannel().createPaintableLayer("Test")
>>> mari.patch_links.linkPatches([patch_1001,patch_1011],"Test Link",layer.imageSet())
>>> print mari.patch_links.groupNamesForImageSet(layer.imageSet())[0]
Test Link

Method Details

canLinkImageInternals ( self , Images )

source code  

Checks whether the input images can be linked based on their internal structure.

Images can be linked if

  • they have the same dimensions,
  • they have the same pixel format,
  • they belong to the same ImageSet and
  • the image set is not a Ptex image set.
Parameters:
  • Images (list of Image )
Returns: bool

canLinkImages ( self , Images , GroupName )

source code  

Checks whether the input images can be linked into an existing/new group with the specified group name.

This will check whether the input images, and any existing images linked in the group indexed by the specified groupname, can have their internals linked. It also checks whether any of the images are already part of other link groups, in which case they can not be linked, and this method will return False.

Parameters:
  • Images (list of Image ) - Container of input Image objects.
  • GroupName (str) - The name for the group in which the images need to be linked.
Returns: bool
True if the images can be linked in a group with the specified group name, and False if not.
Raises:
  • RuntimeError - Raised if any of the input images are not valid.
  • ValueError - Raised if GroupName is an empty string.

See Also: canLinkImageInternals()

canLinkPatchInternals ( self , Patches , pImageSet )

source code  

Checks whether the input patches for the specified image set can be linked based on their underlying image's internal structure.

This is a convenience function which will look up the underlying images for the patches (similar to what can be done by using method mari.GeoEntity.patchImage), and then checks whether the image internals can be linked.

Parameters:
  • Patches (list of GeoPatch ) - Container of input GeoPatch objects.
  • pImageSet ( ImageSet ) - The ImageSet object containing the images for the patches.
Returns: bool
True if the patches internals can be linked, or False if not.
Raises:
  • RuntimeError - Raised if any of the input patches are not valid.
  • ValueError - Raised if pImageSet isn't a valid ImageSet .

See Also: canLinkImageInternals() , mari.GeoEntity.patchImage()

linkImages ( self , Images , GroupName )

source code  

Link the input images into a new/existing group with the specified group name.

This method will check the images are linkable by using canLinkImages() , and if so, it will link them. If the images are linked into an existing group, their contents will immediately be updated to match that of other images in the link group.

Parameters:
  • Images (list of Image ) - Container of Image objects to be linked into an existing/new group.
  • GroupName (str) - A name string indexing an existing group, or a name for a new group.
Returns: None
Raises:
  • RuntimeError - Raised if the images could not be linked.
  • ValueError - Raised if Images is not a list of valid image objects, or if GroupName is an empty string.

See Also: canLinkImages() , generateUniqueGroupName()
