load ( self , Path , Options = [ ] , CameraNames = None )

source code  

Loads a projector from the given file path.

Requests for projector loading are routed to the correct plugin based on file extension.

The following is an example of loading a camera from an alembic file with some additional camera options: 

>>> import mari
>>> filePath = "/tmp/test.abc"
>>> options = ["FrameOffset = 1", "StartFrame = 2", "EndFrame=4"]
>>> camerasToLoad = ["/CameraTopPersp/CameraTopPersp"]
>>> mari.projectors.load(filePath, options, camerasToLoad)
Parameters:
  • Path (str) - The path to the file to be loaded. A file may contain multiple projectors.
  • Options (list of str) - This is an optional parameter which can be used to pass options to the file loader. Options should be in name=value format. Some examples are StartFrame, EndFrame and FrameOffset.
  • CameraNames (variant) - This is an optional list of cameras to load from the file. Each item of the list is a string, corresponding to the name of the camera in the source file.
Returns: list of Projector
A list of the projectors loaded from the given file, empty list on failure
Raises:
  • RuntimeError - Raised if no project is currently open.
  • ValueError - Raised if the options list contains invalid items.
