isShaderTransformAccurate ( self , Config , InputColorspace , OutputColorspace , LutSize )

source code  

Returns True if the given transform is accurate when run in shaders on the GPU.

By default False is returned if the transform requires a 3D LUT on the GPU, and True if no 3D LUT is required. Users can override the default and set the minimum 3D LUT size required for it to be accurate using the 'setShaderTransformAccuracy' method.

Parameters:
  • Config (str) - The name of a standard OCIO config file or the path to a custom one.
  • InputColorspace (str) - The colorspace of the input image
  • OutputColorspace (str) - The colorspace of the output image
  • LutSize (int) - The size of the 3D LUT texture used on the GPU.
Returns: bool
Whether the corresponding colorspace transform will be accurate when run in shaders on the GPU.
Raises:
  • ValueError - Raised if the OCIO config file or either the input and output colorspaces are invalid.

setShaderTransformCode ( self , Config , InputColorspace , OutputColorspace , Code )

source code  

Sets the associated GLSL source code to use, for the given transform, when run on the GPU.

This overrides the code returned by the OpenColorIO library and can be used to give a more optimal transform. 

>>> # This example overrides the 'sRGB' to 'linear' transform to avoid the use of the standard 3D LUT texture and give better accuracy
>>> mari.ocio.setShaderTransformCode(
>>> 'nuke-default',
>>> 'sRGB',
>>> 'linear',
>>> '''
>>> vec3 v1 = #Input.rgb;
>>> bvec3 mask = greaterThan(v1, vec3(0.04045));
>>> vec3 v2 = v1 / vec3(12.92);
>>> v1 = pow((v1 + vec3(0.055)) / vec3(1.055), vec3(2.4));
>>> #Output = vec4(mix(v2, v1, mask), #Input.a);
>>> ''')
Parameters:
  • Config (str) - The name of a standard OCIO config file or the path to a custom one.
  • InputColorspace (str) - The colorspace of the input image
  • OutputColorspace (str) - The colorspace of the output image
  • Code (str) - The GLSL source code to use for the transform on the GPU.
Returns: None
Whether the corresponding colorspace transform will be accurate when run in shaders on the GPU.
Raises:
  • ValueError - Raised if the OCIO config file or either the input and output colorspaces are invalid. Also if the shader input or shader output are missing from the given GLSL source code.

source code
