Class GLRender

source code

Manages Mari's OpenGL Renderer and associated objects.

Mari uses OpenGL to render in 3D. This object gives access to this renderer, allowing configuration of available shaders and other settings.

Shader modules can be applied using the appropriate methods on GeoEntity objects.

Example Code 

>>> # This example checks if color manager is applied
>>> import mari
>>> is_color_manager_ppplied = mari.gl_render.isPostProcessingEnabled()

createQuickApplyGLSL ( self , Name , DefinitionsSnippet = ' ' , BodySnippet = ' ' , IconPath = ' ' , DisplayMetadata = True )

source code  

Creates a new GLSL post processing filter that can be applied to the viewport.

Mari supports post-render processing effects. These can be used to color-correct, blur, filter or warp the viewport display to achieve different effects.

Filters are implemented as two snippets of GLSL shader code:

  • The DefinitionsSnippet defines uniforms and functions required for the filter
  • The BodySnippet contains the implementation of the filter itself

As multiple instances of a filter can be created, we need to give each instance a unique ID. To do this, all uniforms and function names in the shader should have a _$ID_ appended to their names. Mari takes this token internally and adds a unique ID. This preventing name collisions in shaders. For example, a definitions snippet that looks like:

uniform float ScaleX_$ID_;

uniform float ScaleY_$ID_;

will be translated internally into:

uniform float ScaleX1;

uniform float ScaleY2;

Body snippets should be protected with braces, and can expect the color value of the pixel being processed to be available in a vec4 called "Out".

If you wanted to simply multiply the viewport display pixels by a floating point value, for example, you could write something like:

definition_snippet = "uniform float ScaleValue_$ID_;"

body_snippet = "{ Out.rgb *= ScaleValue_$ID_; }"

The values passed into the shader can be controlled by modifying a metadata value on the PostFilter object. The metadata value should have the same name and type as the uniform, but should not include the _$ID_. In the scaling example above, we could add some metadata to the filter called ScaleValue, and this would then be passed into the shader automatically.

Parameters:
  • Name (str) - The name for the new filter
  • DefinitionsSnippet (str) - A snippet of GLSL code that defines the uniforms and functions used by this post process filter
  • BodySnippet (str) - A snippet of GLSL code that defines the body of the post process filter
  • IconPath (str) - Path of the icon used to represent the filter with
  • DisplayMetadata (bool) - If set to True the dialog will expose the metadata set on the filter allowing users to manipulate it
Returns: GLSLFilter
A new post process filter object
Raises:
  • ValueError - Raised if a filter of the given name already exists.

registerCustomBlendMode ( DisplayName , FunctionName , BlendModeCode )
Static Method

source code  

Registers a new custom blend mode.

Registers a new custom blend mode written in GLSL. The code must contain a function :

  • that returns the result of blending in vec4
  • that takes in three arguments where the first argument is the base value in vec4, the second argument is the blended value in vec4, and the third argument is the blend amount
  • that has the function name matching the FunctionName parameter

Example Code 

>>> # This example registers a custom blend mode
>>> mari.gl_render.registerCustomBlendMode("My Mix", "Comp_MyMix",'''
... vec4 Comp_MyMix(vec4 base, vec4 over, float blend_amount)
... {
...     //Blends using standard OpenGL mix for RGB with the blend_amount while the alpha is the max of base or over
...     return vec4( mix(base.rgb, over.rgb, blend_amount), max(base.a,over.a) );
... }
... ''')
Parameters:
  • DisplayName (str) - The name of the blend mode to display in the GUI.
  • FunctionName (str) - The name of the GLSL function to call.
  • BlendModeCode (str) - The blend mode logic written in GLSL.
Returns: None

registerCustomCodeFile ( UUID , FilePath )
Static Method

source code  

Registers a new custom code file written in GLSL.

Registers a new custom code file written in GLSL. The text contents of the given file will be included in the compilation of Mari's GLSL shader. In combination with the header file registered with registerCustomHeaderFile() , the functions defined in the registered file will be available to custom layers and shaders. This behaves like .c file of C language. The UUID uniquely identifies the file. Another path registered with the same UUID is regarded as a new version for the file and replace the old one. The UUID and path pair is also used to track missing files.

Parameters:
  • UUID (str) - Unique ID of the file
  • FilePath (str) - The file path containing the GLSL
Returns: None
Raises:
  • IOError - Raised if the given file does not exist

runGLSLCode ( self , GLSLCode , Destination , Source = None , Mask = None , BlendMode = ' ' , BlendAmount = 1.0 )

source code  

Runs the given GLSL code and writes the result into the given ImageSet .

The GLSL code must write the result into the "Output" variable as vec4.

When Source is provided, the GLSL code can use the values from the texture values in Source by using the token, "#Input". The result will be blended onto the Source ImageSet with Mask applied using the BlendMode and BlendAmount

Examples :

This fills selected area in red and leave the rest black. mari.gl_render.runGLSLCode("Output = vec4(_State.Selected,0,0,1);",DestImageSet)

This fills the selected area in red and blends it with Source image set and writes the result into Dest mari.gl_render.runGLSLCode("Output = vec4(1,0,0,_State.Selected);",DestImageSet,SourceImageSet)

This amplifies the red component of non-masked area of the Source imageset and writes the result into Dest mari.gl_render.runGLSLCode("Output = vec4(#Input.r*2.0,#Input.gba",DestImageSet,SourceImageSet,MaskImageSet)

This fills the selected area in transparency. Note that, in order to modify alpha component, you must use Comp_Copy blend mode mari.gl_render.runGLSLCode("Output = mix(#Input, vec4(0,0,0,0), float(_State.Selected));",DestImageSet,SourceImageSet,None,"Comp_Copy")

Parameters:
  • GLSLCode (str)
  • Destination ( ImageSet )
  • Mask ( ImageSet )
  • Source ( ImageSet )
  • BlendMode (str)
  • BlendAmount (float)
Returns: None
