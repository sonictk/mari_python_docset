This is the interface for creating, traversing and destroying layers on any part of the layer system that has layer stacks. Obtain these using methods on the relevant interfaces that return LayerStack . For example: layer.makeMaskStack() Example Code

>>> current_layer = mari.geo.current().currentChannel().currentLayer()
>>> mask_stack = current_layer.makeMaskStack()

exportImages ( self , PathAndTemplate , Options = 0 , UVIndexList = [ ] , FileOptions = None )

source code  

Exports a set of images from the layer stack.

This is the same as selecting "Export All Layers" from the GUI within the given layer stack. To export an individual layer, call PaintableLayer.exportImages() or ProceduralLayer.exportImages() .

Pass in the path and template to export the files to - for example: 

>>> import mari
>>> channel = mari.geo.current().currentChannel()
>>> channel.exportImages('/tmp/$CHANNEL.$LAYER.$UDIM.tif')

Pass in the path and template to export the files to and two options - for example: 

>>> import mari
>>> channel = mari.geo.current().currentChannel()
>>> channel.exportImages('/tmp/$CHANNEL.$LAYER.$UDIM.tif', 0, [], {'A' : 'B', 'C' : 'D'})
Parameters:
  • PathAndTemplate (str) - The target path for the export and the template to export the files to, in a single string.
  • Options (int) - Pass in a value other than the default to turn on an option, such as disabling "small uniforms". See Image.SaveOptions .
  • UVIndexList (list of int) - Pass in a list of zero-based UV indices if desired to specify the patches to export images for. If not specified or empty, images will be exported for all indices. The UDIM for a patch is the zero-based UV index plus 1001.
  • FileOptions (variant) - This is a key/value map of optional parameters passed to the file writer plugin
Returns: None
Raises:
  • ValueError - Raised if the path and template parameter was empty, or if an invalid UV index was specified.
  • RuntimeError - Raised if the stack contains multiple layers, but the export was not to a layered format and the template did not contain $LAYER. This would otherwise write the same file multiple times.
  • IOError - Raised if an error occurred with the export process.

See Also: exportImagesFlattened() , PaintableLayer.exportImages() , ProceduralLayer.exportImages() , importImages()

exportImagesFlattened ( self , PathAndTemplate , Options = 0 , UVIndexList = [ ] , FileOptions = None )

source code  

Exports a set of images from the flattened contents of the layer stack.

This is the same as selecting "Export All Layers Flattened" from the GUI within the given layer stack. It bakes the contents of the layer stack into a single temporary image for each patch, and then exports those.

Pass in the path and template to export the files to - for example: 

>>> import mari
>>> channel = mari.geo.current().currentChannel()
>>> channel.exportImagesFlattened('/tmp/$CHANNEL.$UDIM.tif')

Some file specific options can be passed via FilOptions - for example : 

>>> import mari
>>> channel = mari.geo.current().currentChannel()
>>> channel.exportImagesFlattened('/tmp/$CHANNEL.$UDIM.exr',0,[],{"compression":"zip"})

For OpenEXR files, FileOptions uses the "compression" method as an option for selecting which compression type is used. The following values are accepted as OpenEXR "compression" options.

  • 'none' : 'none'
  • 'Zip (1 scanline)' : 'zips'
  • 'Zip (16 scanlines)' : 'zip'
  • 'PIZ Wavelet (32 scanlines)' :'piz'
  • 'RLE' : 'rle'
  • 'B44': 'b44'
  • 'B44A': 'b44a'
  • 'DWAA': 'dwaa'
  • 'DWAB': 'dwab'

Note that both the long and short names for the compression method are accepted. Either of the following lines will set the compression type to Zip (16 scanlines). 

>>> channel.exportImagesFlattened('/tmp/$CHANNEL.$UDIM.exr',0,[],{"compression":"zip"})
>>> channel.exportImagesFlattened('/tmp/$CHANNEL.$UDIM.exr',0,[],{"compression":"Zip (16 scanlines)"})
Parameters:
  • PathAndTemplate (str) - The target path for the export and the template to export the files to, in a single string.
  • Options (int) - Pass in a value other than the default to turn on an option, such as disabling "small uniforms". See Image.SaveOptions .
  • UVIndexList (list of int) - Pass in a list of zero-based UV indices if desired to specify the patches to export images for. If not specified or empty, images will be exported for all indices. The UDIM for a patch is the zero-based UV index plus 1001.
  • FileOptions (variant) - This is a key/value map of optional parameters passed to the file writer plugin
Returns: None
Raises:
  • ValueError - Raised if the path and template parameter was empty, or if an invalid UV index was specified.
  • IOError - Raised if an error occurred with the export process.

See Also: exportSelectedPatches() , exportImages() , importImages() , exportSelectedPatchesFlattened()

hash ( self , UVIndex = -1 )

source code  

Returns a unique identifier for this layer stack.

This hash is assumed to be universally unique (256bit skein hash). This can be used to determine if a layer stack or its children have changedsince some given checkpoint. A different hash means a different layer or children. This hash is based on image data, blend mode, parameters and anything that might effect the pixel output. Use this to detect any changes to the layers.

Parameters:
  • UVIndex (int) - The UV index of the patch to return the hash of, or -1 to return the hash of all the patches.
Returns: str
A hash string that uniquely identifies this version of this layer stack and its childrens, or an empty string on failure.

See Also: imageHash()

imageHash ( self , UVIndex = -1 )

source code  

Returns a unique identifier for this layer stack's image data.

importImages ( self , PathAndTemplate , ScaleOption = 1 , MatchingLayersOption = 0 , MismatchingLayersOption = 0 , UVIndexList = [ ] , RemoveAlpha = False )

source code  

Imports a set of images into the layer stack.

This is the same as selecting "Import into Layer Stack" from the GUI. To import into an individual layer, call PaintableLayer.importImages() .

Pass in the path to the files and the template to import them from - for example: 

>>> channel.importImages('/tmp/$CHANNEL.$UDIM.psd')
Parameters:
  • PathAndTemplate (str) - The path to the files and the template to import them from, in a single string.
  • ScaleOption ( ImageSet.ScaleChoice ) - When the existing patch and the image in the file to import are of different sizes, this indicates which of the two to rescale to match the other
  • MatchingLayersOption ( LayerImportOption ) - Determines what to do when the current layer stack and the layers being imported contain matching layer names, where the types of both layers match. This is the most common case. The default is to update the existing layers.
  • MismatchingLayersOption ( LayerImportOption ) - Determines what to do when the current layer stack and the layers being imported contain matching layer names, where the types of the existing layers and those being imported do not match. This can happen when an exported file contains procedural layers baked as paintable, for example. The default is to skip the mismatching layers.
  • UVIndexList (list of int) - Pass in a list of zero-based UV indices if desired to specify the patches to export images for. If not specified or empty, images will be exported for all indices. The UDIM for a patch is the zero-based UV index plus 1001.
  • RemoveAlpha (bool) - Remove the alpha channel (convert RGBA to RGB).
Returns: None
Raises:
  • ValueError - Raised if the path and template parameter was empty, or if one or both of the matching and mismatching layers options was invalid.
  • IOError - Raised if an error occurred with the import process.

See Also: exportImages() , PaintableLayer.importImages()

  • GroupLayersToUngroup (list of Layer ) - The list of group layers to ungroup. If empty all currently selected layers will be used.
Returns: None
Raises:
  • ValueError - Raised if the layers to ungroup are not in this stack.
