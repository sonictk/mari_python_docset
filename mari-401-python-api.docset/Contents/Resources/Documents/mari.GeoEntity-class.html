addVersion ( self , Paths , Name , Options = None )

source code  

Adds a new version of the object from one or more files.

It is often useful to have multiple versions of a geometry available to test.

When adding versions, it is assumed that only the world space positions and normals of the objects have changed.

When dealing with geometry with updated UVs, no conversion occurs, and rendering artifacts may be seen.

As in GeoManager.load() , the settings for the geometry load can be controlled by the function parameter Options , and the format of the parameter is the same.

Parameters:
  • Paths (variant) - The path(s) to the file(s) containing the new version geometry. This must be either a single pathname, or a list of pathnames.
  • Name (str) - The name for the new version. This must be unique for this item.
  • Options (variant) - An optional dictionary (or list, for backwards compatibility) of settings to control the geometry load, as in GeoManager.load()
Returns: None
Raises:
  • ValueError - Raised if no paths were provided, any of the file paths were empty, the version name already exists, or the options string list was invalid.
  • IOError - Raised if any of the files to load the version from was not found.
  • TypeError - Raised if the paths parameter is not a string or list of strings.
  • RuntimeError - Raised if the mesh types are invalid for adding version.

Note: The Options initially default to the settings from the current version of the Object. Any options provided are treated as overrides to the defaults.

Deprecated: Please pass the Options parameter in {'name': value} dictionary format, instead of the old ['name=value'] list format.

See Also: versionList() , removeVersion() , renameVersion() , versionNames()

createChannel ( self , Name , Width , Height , Depth , IsShaderStack = False )

source code  

Creates a new channel from the settings provided.

This function can be used to create either UV or Ptex channels, depending on whether or not the entity is using Ptex . For Ptex channels, the Width and Height parameters are ignored, so using the function createPtexChannel() is more convenient.

Parameters:
  • Name (str) - The name to call the new channel
  • Width (int) - The default width of the channel (ignored for Ptex )
  • Height (int) - The default height of the channel (ignored for Ptex )
  • Depth (int) - The default bit depth of the channel
  • IsShaderStack (bool) - If True create the channel as a ShaderStack. If none is passed a nonShaderStack is created.
Returns: Channel
The newly-created channel
Raises:
  • ValueError - Raised if the channel name is already in use, depth is not supported or width and height are not supported
  • RuntimeError - Raised if the channel could not be created

See Also: createDuplicateChannel() , createPtexChannel()

createShader ( self , Name , Type , SpecularType = ' ' )

source code  

Creates a standalone shader or diffuse and specular shader.

Creates a standalone shader or diffuse and specular shader.

This method is provided for compatibility only and will be removed in a future version.

Please use createStandaloneShader() or createDiffuseSpecularShader() instead.

This creates a standalone shader or a shader consisting of diffuse and specular. For example,

To create a standalone shader of type Flat:

GeoEntity.createShader("Flat","Lighting/Standalone/Flat");

To create a diffuse and specular shader of type Lambertian and Phong:

GeoEntity.createShader("Phong","Lighting/Diffuse/Lambertian","Lighting/Specular/Phong");

Parameters:

  • Type (str) - The standalone type of the shader to create if SpecularType is empty. The diffuse type of the shader to create if SpecularType is provided.
  • SpecularType (str) - The specular type of the shader to create.
Returns: Shader
The new shader.
Raises:
  • RuntimeError - Raised if the shader could not be created.

Deprecated: This method will be removed in a future version. Please use createStandaloneShader() or createDiffuseSpecularShader() instead.

generateSubdivision ( self , Options )

source code  

Generates subdivision surfaces.

This calls OpenSubdiv library to generate subdivision surfaces. Options can be specified to set the following:

  • Level (int) : Subdivision Level
  • Force (bool) : True to force subdivision of geometry pieces marked as "Don't subdivide"
  • Use Source (bool) : True, to automatically use subdivision settings (scheme and rules) from the source geometry file, or False, to use user-supplied settings.
  • Scheme ( GeoManager.OpenSubdivisionSchemeType ) : Subdivision scheme
  • Boundary Interpolation ( GeoManager.OpenSubdivisionBoundaryInterpolation ) : Boundary interpolation rule
  • Face Varying Interpolation ( GeoManager.OpenSubdivisionFaceVaryingInterpolation ) : Face-varying data interpolation rule
  • Creasing ( GeoManager.OpenSubdivisionCreasingMethod ) : Creasing method
  • Triangle Subdivision ( GeoManager.OpenSubdivisionTriangleSubdivision ) : Triangle subdivision rule

The following options are DEPRECATED:

  • Scheme (string) : One of "Catmull Clark", "Loop" or "Bilinear"
  • Boundary Interpolation (string) : One of "None", "Edge Only", "Edge And Corner (No Corner Progation)" or "Edge And Corner (Corner Propagation)" or "Always Sharp"

Examples: 

>>> import mari
>>> # Subdivide using default settings from source file
>>> options = {}
>>> mari.geo.current().generateSubdivision(options)
>>>
>>> # Subdivide using overriden settings, Catmull clark subdivision at level 2
>>> options = {}
>>> options["Level"] = 2
>>> options["Scheme"] = mari.geo.SCHEME_CATMARK
>>> mari.geo.current().generateSubdivision(options)
>>>
>>> # Subdivide using overriden settings, Loop clark subdivision at level 3, boundary interpolation set to edge only, uv face varying interpolation set to boundaries
>>> options = {}
>>> options["Level"] = 3
>>> options["Scheme"] = mari.geo.SCHEME_LOOP
>>> options["Boundary Interpolation"] = mari.geo.VTX_BOUNDARY_EDGE_ONLY
>>> options["Face Varying Interpolation"] = mari.geo.FVAR_LINEAR_BOUNDARIES
>>> mari.geo.current().generateSubdivision(options)
>>>
>>> # Subdivide using overriden settings, Catmull clark subdivision at level 1, creasing set to chaikin and triangle subdivision set to smooth
>>> options = {}
>>> options["Level"] = 1
>>> options["Scheme"] = mari.geo.SCHEME_CATMARK
>>> options["Creasing"] = mari.geo.CREASE_CHAIKIN
>>> options["Triangle Subdivision"] = mari.geo.TRI_SUB_SMOOTH
>>> mari.geo.current().generateSubdivision(options)
Parameters:
  • Options (QMap of str, variant) - Options for generating subdivision surfaces up to.
Returns: bool
Returns whether subdivision succeeded.
Raises:
  • ValueError - Raised if the operation fails, with warnings.

Deprecated: Some of the key-values, passed in the Options map will be removed in future versions.

Geometry may be broken into multiple UV patches. Each UV patch occupies a single integer square in UV space. Each patch may have a name, associated metadata, and locked or hidden or valid status.

As of version 1.4v3, these patches are returned in UV index order - the same order as ImageSet.imageList() . Note however that these indices do not necessarily map directly to UDIMs, as some UDIMs may be unused, and there will not be any corresponding spaces left empty in the array. To be sure of the UDIM of a patch, call the GeoPatch.udim() method.

Returns: list of GeoPatch
A list of the UV patch objects associated with this geometry
Notes:
  • In versions older than 1.4v3, this function does not necessarily return the images in any particular order, or in the same order as ImageSet.imageList() .
  • Only valid patches are returned. To retreive all patches use completePatchList() .
  • This method returns the combined list of all patches from the GeoEntityVersion objects belonging to this GeoEntity . See GeoEntityVersion.patchList()

See Also: completePatchList()
