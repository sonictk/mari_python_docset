setNukeHost ( name , port , pid )

source code  

Sets the details of the Nuke instance for Mari to communicate with.

This function is only intended to be called from Nuke via a socket.

Parameters:
  • name (str) - The host name, or None to clear
  • port (int) - The host port, or None to clear
  • pid (int) - The process ID of the Nuke instance that we are communicating with

sendToNuke ( nuke_cmd , ignore_failure = False )

source code  

Sends a command to Nuke via a socket connection.

The command port in Nuke's script settings needs to be enabled for this to work.

Parameters:
  • nuke_cmd - Nuke python command (string) - does not need to start with "nukeDo:". This may be empty to just verify the connection
Returns: bool
True if the command was sent successfully, or False otherwise

getCamData ( projector )

source code  

Converts Mari camera data into Nuke camera data (Vector to Vector, focal to focal and filmback, etc).

Return a dictionary with name and data keys with translate, rotate, haperture, vaperture, focal. This will only work for ZXY rotation order.

Parameters:
  • projector - A Mari projector

exportProjectors ( export_all = True , use_socket = False )

source code  

Export Mari projector to Nuke either via the the socket connection or to a file (pickled python dictionary).

Parameters:
  • export_all - If True, export all projectors; otherwsise only export the current one
  • use_socket - If True, data will be sent to Nuke via the socket; otherwise a file browser will prompt the user for a path
